TAP：把已有 HTTP 服务代理成 Agent 友好的 CLI

Agent 只表达一次意图，Skill 把探索结果沉淀为适配器；后续任务统一通过 TAP 加载固化 pipeline 执行， 在 HTTP、CDP 和用户浏览器登录态之间形成可复用、可审计、确定性更高的工具路径。

意图层：Agent 使用路径

I从自然语言意图到可执行工具

AAgent 表达意图

需要获取信息、查询配置、读取站点状态或调用已有 HTTP 服务。

目标是描述“要什么”，不在每次任务中重写网页探索和请求细节。

SSkill 创建适配器

侦察站点接口、页面行为、登录要求和返回结构。

把稳定路径写成 adapters/<site>/<command>.js。

CCLI 入口固化

tap <site> <command> --key value 成为后续调用界面。

Agent 之后只需传参数并解析结构化输出。

沉淀层：适配器注册与复用

PAdapter 是意图和执行之间的稳定契约

R适配器声明 推荐复用模式

description
→
args

pipeline
→

pipeline 用 fetch、navigate、intercept、select、map、limit 等步骤描述执行。

L加载优先级

TAP_ADAPTERS_DIR 自定义目录优先，用于项目私有命令。

~/.tap/adapters 用户适配器覆盖内置适配器。

adapters/ 内置示例提供默认能力。

执行层：确定性 pipeline runtime

E一次适配，多次确定性执行

1CLI 编排

bin/cli.js 解析站点、命令、参数和输出格式。

加载命中的 adapter，并把参数交给执行器。

2顺序执行步骤

executePipeline() 让每一步接收上一轮 data。

HTTP、选择、映射、过滤、排序、截断组合为稳定数据流。

3结构化输出

默认 JSON 面向 Agent 解析，table 面向人工查看。

输出列由 adapter 的 columns 固定，减少后处理不确定性。

机制层：HTTP + CDP + 登录态

B代理已有服务，同时保留用户浏览器上下文

直接 HTTP 路径

公开接口或无需登录接口走 fetch。

适合新闻列表、热榜、公开配置和轻量信息获取。

CDP 浏览器路径

TAP_CDP_ENDPOINT 连接 Chrome remote debug。

复用用户 Chrome 的 cookie、会话和登录态。

通过 navigate、evaluate、intercept 抽取页面内数据。

收益层：Agent 友好的确定性工具

O意图与执行分离后的任务闭环

D确定性执行

执行路径由 adapter 固化，减少每次临场浏览网页和推断接口。

同一命令、同一参数形成可重复的工具调用行为。

T降低 token 成本

复杂站点知识沉淀在 adapter，而不是每次塞进上下文。

Agent 只消费命令、参数和结构化结果。

Q提升成功率

复用已经验证过的 pipeline，减少网页变化、登录态处理和选择器探索带来的失败。

适合信息获取、配置查询、内部 HTTP 服务代理和运营状态读取。

1

Agent 入口

调用 CLI，而不是直接操作网页或散装 HTTP 请求。

tap linuxdo news --limit 5 --format json
2

适配器契约

把命令参数、pipeline 和输出列固定成可维护文件。

export default { args, pipeline, columns }
3

浏览器会话

需要登录态时，通过 CDP 进入已有 Chrome 上下文。

TAP_CDP_ENDPOINT=http://localhost:9222
4

结果消费

返回 JSON 或表格，便于 Agent 后续摘要、对比和决策。

printOutput(data, format, columns)
